# Contributing to in-zone ⏱️

We are thrilled that you want to help make **in-zone** the ultimate minimalist workflow workspace! To ensure high-quality code and zero merge conflicts, please follow these guidelines:

## Code of Conduct
* Be respectful, professional, and supportive.
* Prioritize accessibility, clear naming conventions, and performance.

## Branching Strategy
* **Do not commit directly to `main`**.
* Create feature branches named: `feature/your-feature-name` (e.g., `feature/login-system`).
* Create bugfix branches named: `bugfix/issue-description` (e.g., `bugfix/timer-reset`).

## Submission Workflow
1. Pull the latest `main` branch before coding: `git pull origin main`
2. Create your branch locally: `git checkout -b feature/my-new-feature`
3. Commit your changes with descriptive, conventional messages: `git commit -m "feat: Add login validation limits"`
4. Push to your branch: `git push origin feature/my-new-feature`
5. Open a **Pull Request** on GitHub against the `main` branch.
6. Await peer review, complete any requested edits, and resolve conflicts before merging.
